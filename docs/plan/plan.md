# 計画

このファイルは、次に何をするかを相談しながら整理するための場所です。

## 現在の方針

Python バックエンド（推論）と Electron + React フロントエンド（UI）を、ローカル HTTP API（`127.0.0.1:8765`）で接続する構成。詳細な技術メモは [../../CLAUDE.md](../../CLAUDE.md) を参照。

```
Electron (React UI) ──HTTP──> FastAPI (生成キュー) ──> engine ──> Stable Audio 3 Medium (CUDA)
```

- 生成は単一ワーカースレッドで 1 件ずつ順次処理（GPU が 1 基のため）
- ジョブと結果は `data/`（WAV + `jobs.json`）に永続化
- Python は `runtime/`（3.10 standalone）+ `.venv` でプロジェクト内蔵

## 優先順位

1. **動くこと・壊さないこと** — 生成パイプラインと永続化の安定
2. **使い勝手の UI** — 生成条件の入力、結果カードの操作（再生・DL・条件コピー）
3. **見た目の統一** — ダークテーマ、自前コンポーネント（スライダー・オーディオプレーヤー）
4. その後に高度機能・配布

## 設計上の決めごと

- 破壊的操作（ファイル削除など）は事前確認を分けて実行する
- 依存は固定バージョンで管理し、Blackwell GPU 向けに PyTorch は cu128 を使う
- UI は素の HTML/CSS + React。重い UI ライブラリは入れない
